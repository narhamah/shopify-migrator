BUG-1  compare_stores.py crashes on Windows (cp1252 encoding)
       Line 1020: json.dump() writes arrow chars to file opened without
       encoding='utf-8'. Same bug in audit_store.py line 946.
       STATUS: FIXED — added encoding='utf-8' to both files.

BUG-2  compare_stores.py reports 118 false-positive "missing" items
       Root cause: script compared Spanish handles (e.g. /champu-hidratante)
       against the Saudi store which uses English handles (/nurture-shampoo).
       Affected: products (41 false missing), collections (54 false missing),
       pages (4 false missing), articles (3 false missing), ingredients (12
       false missing), navigation menus (14 false missing).
       STATUS: FIXED — added Spain→English handle mapping using local data
       files (data/spain_export/ and data/english/ share IDs for 35 products,
       55 collections, 8 pages, 3 articles, 34 ingredients).

BUG-3  Navigation menu comparison fails on /en/ prefix URLs
       Spain menu URLs use /en/ prefix (e.g. /en/pages/quiz), Saudi doesn't.
       Path comparison was exact-match, never finding Saudi equivalents.
       STATUS: FIXED — normalize paths by stripping /en/ prefix before compare.
